### First-time / after-reboot setup

The local Supabase DB is provisioned from `scripts/dev-local-schema.sql`, which is
derived from the context-only dump `sql/1. optigo.sql` (see that file's header for
the transformations: identifiers quoted to preserve PascalCase table names, forward
FKs moved to `ALTER TABLE`, `ARRAY`â†’`text[]`, column-reference `DEFAULT`sâ†’generated
columns, plus post-dump additive columns and role grants; RLS is intentionally not
recreated). One command does everything (start Docker if needed, `supabase start`,
write `.env.local`, load the schema):

```bash
bash scripts/setup-local-supabase.sh
npm run dev
```

`.env.local` is git-ignored and is regenerated by that script from `supabase status`.
If Docker isn't running: `sudo dockerd &` then `sudo chmod 666 /var/run/docker.sock`.
Docker 29 here uses the `fuse-overlayfs` storage driver with the containerd
snapshotter disabled (`/etc/docker/daemon.json`).

### Gotchas

- Do NOT run `npm run build` while `npm run dev` is running â€” both use the same
  `.next` directory and a concurrent build corrupts the dev server (500s). Stop dev
  first, or build in a separate checkout.
- `scripts/dev-local-schema.sql` reruns are DESTRUCTIVE (it drops and recreates the
  `public` schema). Reloading it does NOT delete `auth.users`, so after a schema
  reset a previously-registered email still exists in Auth but its tenant/profile/role
  rows are gone. To start clean, delete the stale auth user first:
  `docker exec -i supabase_db_<project> psql -U postgres -d postgres -c "delete from auth.users where email='demo@optigo.test';"` then re-register.
- The dump `sql/1. optigo.sql` is slightly older than the current app code, so a few
  post-dump tables/columns from `sql/V0xx_*.sql` migrations may be missing. When a
  page 500s on an "unknown column/relationship", add the missing column via
  `ALTER TABLE ... ADD COLUMN IF NOT EXISTS ...` then `NOTIFY pgrst, 'reload schema';`
  (already done for `ChoKham.done_at` / `ChoKham.check_in_source`).
- There is no automated test suite (no `test` script / test runner configured).
- `npm run lint` reports some pre-existing lint errors in app code; builds are
  unaffected because `next.config.js` sets `eslint.ignoreDuringBuilds` and
  `typescript.ignoreBuildErrors`.
